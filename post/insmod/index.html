insmod

2022-02-01

将给定的模块加载到内核中。

补充说明

insmod 命令用于将给定的模块加载到内核中。Linux 有许多功能是通过模块的方式，在需要时才载入 kernel。如此可使 kernel 较为精简，进而提高效率，以及保有较大的弹性。这类可载入的模块，通常是设备驱动程序。

语法

insmod (选项) (参数)

选项

-f：不检查目前 kernel 版本与模块编译时的 kernel 版本是否一致，强制将模块载入；
-k：将模块设置为自动卸除；
-m：输出模块的载入信息；
-o<模块名称>：指定模块的名称，可使用模块文件的文件名；
-p：测试模块是否能正确地载入 kernel；
-s：将所有信息记录在系统记录文件中；
-v：执行时显示详细的信息；
-x：不要汇出模块的外部符号；
-X：汇出模块所有的外部符号，此为预设置。

参数

内核模块：指定要加载的内核模块文件。

实例

加载 RAID1 阵列级别模块，如下所示：

[root@localhost boot]# insmod /lib/modules/2.6.
18-8.el5/kernel/drivers/md/raid1.ko

[root@localhost boot]# lsmod | grep raid1
raid1                  25153  0

从以上显示结果可知，RAID1 模块已加载成功。只是在使用 insmod 命令加载模块时，需要使用绝对路径方能加载，且加载时无法自动解决依赖关系。
